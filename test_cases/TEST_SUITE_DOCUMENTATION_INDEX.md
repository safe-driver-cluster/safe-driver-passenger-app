## 🎓 Key Concepts

### **Test Case Format**

Each test case follows this standard format:

```
✅ TC-XXX-XXX: [Clear Test Title]

📖 Description:
   [What are we testing and why?]

🔧 Preconditions:
   [What must be true before test starts?]

📌 Test Steps:
   1. [Step 1]
   2. [Step 2]
   [... 10-12 steps total]

✅ Expected Result:
   [What should happen if test passes?]

☑️ Result: [PASS / FAIL / BLOCKED]
```

### **Severity Levels** (Priority)

| Severity | Impact | Example |
|----------|--------|---------|
| **CRITICAL (P0)** | Blocks entire app | App won't start, all logins fail |
| **HIGH (P1)** | Major feature broken | Feedback can't be submitted |
| **MEDIUM (P2)** | Minor issue | Button has wrong color |
| **LOW (P3)** | Cosmetic or edge case | Typo in error message |

### **Test Status Codes**

| Status | Meaning | Action |
|--------|---------|--------|
| ✅ **PASS** | Test successful | Move to next test |
| ❌ **FAIL** | Test failed | Log bug, mark for retest |
| ⏸️ **BLOCKED** | Can't run test | Note blocker, try later |
| ⚠️ **WARNING** | Passed with notes | Document warning |
| ⏳ **N/A** | Not applicable | Skip, mark N/A |
